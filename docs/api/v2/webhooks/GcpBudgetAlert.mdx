title: "GCP budget alert webhook"
description: "Receive Google Cloud budget threshold alerts and fan them out via WhatsApp."

Inbound webhook that receives budget-threshold notifications from a Google Cloud Function (`budget-alert-handler`) and forwards `ERROR` / `CRITICAL` events to a hardcoded list of WhatsApp recipients using the `alerta_sistema` template. `WARNING` events are logged but not forwarded.

This endpoint is **internal to Velip** — it exists to keep ops on call when GCP costs spike. It is documented here only for completeness; third parties have no reason to call it.

## Authentication

This endpoint **does not** use `tsid`. Instead it requires a shared-secret header:

| Header | Value |
| --- | --- |
| `X-Webhook-Secret` | The value of the `GCP_BUDGET_WEBHOOK_SECRET` credential (rotated together with the Cloud Function). |

The comparison is constant-time (`hash_equals`). If the secret is not configured server-side, the endpoint returns `503 Webhook not configured` (fail-closed).

## Request

<ParamField body="source" type="string" required>
  Must be `gcp-cost-guard`. Anything else returns `400`.
</ParamField>

<ParamField body="severity" type="string" required>
  `WARNING` (logged only) | `ERROR` | `CRITICAL` (both forwarded by WhatsApp).
</ParamField>

<ParamField body="budget_name" type="string" required>
  Budget label as configured in GCP.
</ParamField>

<ParamField body="summary" type="string">
  Free-text summary that becomes the WhatsApp message body.
</ParamField>

<ParamField body="cost" type="number" required>
  Current cost in BRL (formatted as `R$ X,XX` before forwarding).
</ParamField>

<ParamField body="budget" type="number" required>
  Configured budget in BRL.
</ParamField>

<ParamField body="percentage" type="string" required>
  Percentage of budget consumed (e.g., `80.0%`).
</ParamField>

<ParamField body="action" type="string">
  Optional action hint sent by the Cloud Function.
</ParamField>

<ParamField body="timestamp" type="string">
  ISO-8601 timestamp of the alert.
</ParamField>

## Behaviour

1. Always inserts a row in `cd_log_alert` with the alert details.
2. Anti-flood: if an alert with the same `cdlal_name` and `cdlal_param_1` was logged in the last 60 minutes, no WhatsApp is sent.
3. Otherwise, fires a `MakeWhatsapp` template (`alerta_sistema`, `pt_BR`) to each recipient in `A_dest` (hardcoded ops phones).

## Status codes

| HTTP | Meaning |
| --- | --- |
| `200` | Alert accepted (whether or not a WhatsApp was forwarded). |
| `400` | Invalid payload (`source` mismatched, JSON parse error). |
| `403` | `X-Webhook-Secret` missing or wrong. |
| `405` | Method not `POST`. |
| `500` | Database connection error. |
| `503` | The webhook secret is not configured server-side. |

</Note>
