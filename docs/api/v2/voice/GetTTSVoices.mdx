---
title: "List TTS voices"
description: "Return TTS voices the authenticated account is allowed to use."
api: "POST https://<base>/api/v2/GetTTSVoices.php"
---

Returns the list of TTS voices available to the authenticated customer. The list is filtered by the `cdwv_aut_cdcs_id` column on `cd_wav_voices` — voices flagged for `0` (universal) or for this specific `cdcs_id` are returned.

Use the `cdwv_company` + `cdwv_short` (or `cdwv_name`) values to compose the `voice` parameter for [`MakeTTSCall`](/docs/api/v2/voice/MakeTTSCall) and [`CreateAudioFile`](/docs/api/v2/audio-files/CreateAudioFile).

## Authentication

Token authentication required. See [Authentication](/docs/api/v2/authentication).

## Request

<ParamField body="tsid" type="string" required>
  Token for the account.
</ParamField>

This endpoint takes no other parameters.

## Request example

<RequestExample>

```bash curl
curl -X POST 'https://<base>/api/v2/GetTTSVoices.php' \
  -H 'Content-Type: application/json' \
  -d '{ "tsid": "YOUR_TSID" }'
```

</RequestExample>

## Response

<ResponseExample>

```json 200 OK
{
  "return": {
    "status": "OK",
    "status_code": "0"
  },
  "voices": [
    {
      "cdwv_id": 12,
      "cdwv_name": "google|pt-BR-Wavenet-A",
      "cdwv_tit": "Camila",
      "cdwv_short": "pt-BR-Wavenet-A",
      "cdwv_type": "tts",
      "cdwv_language": "pt-BR",
      "cdwv_genre": "F",
      "cdwv_company": "google",
      "cdwv_sample": "https://..."
    }
  ]
}
```

</ResponseExample>

<ResponseField name="voices[].cdwv_name" type="string">
  Voice id in the format `provider|VoiceName` — pass it directly as the `voice` parameter in [`MakeTTSCall`](/docs/api/v2/voice/MakeTTSCall).
</ResponseField>

<ResponseField name="voices[].cdwv_sample" type="string">
  URL to a short MP3 sample of the voice, useful for previewing.
</ResponseField>

<ResponseField name="voices[].cdwv_genre" type="string">
  Voice gender hint: `F` (female), `M` (male).
</ResponseField>

## Error codes

This endpoint inherits the [global authentication codes](/docs/api/v2/errors). It does not raise endpoint-specific business codes.

<Note>
ElevenLabs voices may appear in this list (they are valid for [`CreateAudioFile`](/docs/api/v2/audio-files/CreateAudioFile)) but are blocked by [`MakeTTSCall`](/docs/api/v2/voice/MakeTTSCall) — using them in `MakeTTSCall` returns code `210`.
</Note>
