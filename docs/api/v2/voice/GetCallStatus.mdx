---
title: "Get call status / list calls"
description: "Query call records (single or paginated) and optionally push them to a webhook."
api: "POST https://<base>/api/v2/GetCallStatus.php"
---

Returns call records — a single call when you pass `cd_id` or `ctid`, or a paginated list when you pass campaign / date filters. When you supply `rurl`, results are POSTed to your webhook in either JSON or URL-encoded format instead of being returned in the response body.

This is the polling counterpart to the per-account return URL configured via the Velip portal.

## Authentication

Token authentication required. See [Authentication](/docs/api/v2/authentication).

## Request

<ParamField body="tsid" type="string" required>
  Token for the account.
</ParamField>

<ParamField body="cd_id" type="string">
  Single call id, in the form `<cdcs_db>_<numeric>` returned by [`MakeTTSCall`](/docs/api/v2/voice/MakeTTSCall). When supplied, returns one record.
</ParamField>

<ParamField body="ctid" type="string">
  Customer-side correlation id. Pass `null` to query records with empty `ctid`.
</ParamField>

<ParamField body="cp_id" type="integer">
  Campaign id (`cd_programa.cp_id`). Returns all calls for the campaign.
</ParamField>

<ParamField body="cpid" type="string">
  Customer-side campaign id (`cd_programa.cp_ctid`). Resolved internally to the latest matching `cp_id`.
</ParamField>

<ParamField body="date_start" type="string">
  Start date (`YYYY-MM-DD`). Alias `dini`.
</ParamField>

<ParamField body="date_end" type="string">
  End date (`YYYY-MM-DD`). Alias `dend`.
</ParamField>

<ParamField body="maxreg" type="integer" default="1000">
  Maximum records returned per page. Pass `-1` for unlimited (use with care). The endpoint actually fetches `maxreg + 1` rows so it can detect whether more pages exist.
</ParamField>

<ParamField body="last_id" type="integer">
  Cursor for pagination. Pass the `pagination.next_id` of the previous response.
</ParamField>

<ParamField body="onlydtmf" type="integer">
  When `1`, only return calls where the user pressed at least one digit (any of `cd_resp_1`, `cd_resp_2`, `cd_resp_3` non-empty).
</ParamField>

<ParamField body="numtype" type="string">
  Format for the destination number in the response. Empty = `DDI+DDD+number`, `0DDD` = `0+DDD+number`, `DDD` = `DDD+number` (Brazilian numbers only).
</ParamField>

<ParamField body="origin" type="string" default="api">
  Marker echoed back in `origin`. Allowed `[a-zA-Z0-9_-]{1,32}`; otherwise reset to `api`.
</ParamField>

### Webhook delivery

Pass `rurl` to deliver the results to your endpoint instead of receiving them inline. Per-account defaults are used when these fields are omitted.

<ParamField body="rurl" type="string">
  Public URL that receives the data.
</ParamField>

<ParamField body="rurl_method" type="string">
  `POST` or `GET`. Default is the customer's `cdcs_return_url_type`.
</ParamField>

<ParamField body="rurl_type" type="string">
  `json` to deliver as a JSON body; otherwise URL-encoded form parameters per call. Default is the customer's `cdcs_return_data_type`.
</ParamField>

<ParamField body="rurl_headers" type="string">
  Newline-separated extra headers (`Header: value`).
</ParamField>

<ParamField body="rurl_user" type="string">
  HTTP Basic username for the webhook.
</ParamField>

<ParamField body="rurl_password" type="string">
  HTTP Basic password for the webhook.
</ParamField>

<ResponseField name="calls[].cd_amdasw" type="string">
  Answering-machine detection result: `MA` (machine), `HM` (human), `ND` (not determined). Only meaningful when AMD was enabled on the campaign.
</ResponseField>

<ResponseField name="calls[].resp_1, resp_2, ..." type="string">
  DTMF responses captured during the call. Number of `resp_*` keys is dynamic — depends on how many digits the IVR collected.
</ResponseField>

<ResponseField name="calls[].cdr_url" type="string">
  Recording URL when recording was enabled. `cdr_sec` carries the duration in seconds.
</ResponseField>

<ResponseField name="return.pagination.has_more" type="boolean">
  Present only when `maxreg` is set. `true` when more pages exist.
</ResponseField>

<ResponseField name="return.pagination.next_id" type="integer">
  Cursor to pass as `last_id` on the next page.
</ResponseField>

## Error codes

This endpoint inherits the [global authentication codes](/docs/api/v2/errors). It does not raise endpoint-specific business codes — invalid filters simply produce an empty `calls` list.

<Note>
This endpoint is read-only on the call records (`cd_envia_*`), but writes to `cd_returl` when `rurl` is supplied — to track whether the webhook delivery succeeded. `cd_returl > 0` = delivered, `< 0` = retries failed.
</Note>
