---
title: "List campaigns"
description: "Search and list campaigns belonging to the authenticated account."
api: "POST https://<base>/api/v2/GetCampaignsList.php"
---

Returns up to `maxreg` campaigns sorted by descending id, with current schedule, throughput, and progress counters. Filters let you scope by group, ctid, single id, model templates, end-date floor, or free-text search.

## Authentication

Token authentication required. See [Authentication](/docs/api/v2/authentication).

## Request

<ParamField body="tsid" type="string" required>
  Token for the account.
</ParamField>

<ParamField body="cp_id" type="integer">
  Single campaign id.
</ParamField>

<ParamField body="cp_ctid" type="string">
  Customer-side campaign id (`cp_ctid`).
</ParamField>

<ParamField body="group" type="integer">
  Group id (`cp_cdg_id`).
</ParamField>

<ParamField body="type" type="string">
  Pass `on` to return only currently active campaigns (`cp_ativo = 1`).
</ParamField>

<ParamField body="model" type="string">
  Pass `1` to return only campaigns flagged as model templates (`cp_model_show = 1`).
</ParamField>

<ParamField body="date" type="string">
  Filter by end date. `1` = today onward; otherwise a `YYYY-MM-DD` to floor `cp_data_fim`.
</ParamField>

<ParamField body="search" type="string">
  Free-text search across `cp_nome`, `cp_descritivo`, `cp_id`, and `cm_vmc_audio`.
</ParamField>

<ParamField body="maxreg" type="integer" default="500">
  Maximum number of records to return.
</ParamField>

## Request example

<RequestExample>

```bash curl
curl -X POST 'https://<base>/api/v2/GetCampaignsList.php' \
  -H 'Content-Type: application/json' \
  -d '{ "tsid": "YOUR_TSID", "type": "on", "maxreg": 50 }'
```

</RequestExample>

<ResponseField name="campaigns[].cp_ontime " type="string">
  `1` when the campaign is active **and** the current date/time is inside its schedule window.
</ResponseField>

<ResponseField name="campaigns[].cp_pas" type="string">
  Concurrency limit for transferred calls (`cp_cdtrf_emcurso_lim`). `null` for campaigns whose model is not transfer-based.
</ResponseField>

<ResponseField name="campaigns[].cp_transfered" type="string">
  Number of calls that requested transfer. `null` for non-transfer campaigns.
</ResponseField>
